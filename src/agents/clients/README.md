# Client Plugin Directory

Per-organization plugin directories for custom agents, tools, and workflows.

## Structure

```
clients/
  _example/              ← Reference implementation (not loaded in production)
  {org_id}/
    agents/              ← Custom agent overrides (take priority over specialists)
    tools/               ← Niche-specific tools (no direct Infisical access)
    workflows/           ← Custom YAML workflow templates
    brand-prompt.md      ← Brand System Prompt (generated by Brand Persona Agent)
```

## Resolution Order

When the workflow engine resolves an agent by name:

1. `clients/{org_id}/agents/{agentName}` — client-specific override
2. `platforms/{platform}/subagents/{agentName}` — platform Tier 2.5
3. `specialists/{agentName}` — core Tier 3 generic

## Provisioning

Use the CLI to scaffold a new client plugin directory:

```bash
npx ts-node src/cli/commands/init-plugin.ts --org <org_id>
```

## Rules

- **No direct Infisical access** — credentials are injected by the worker via `prepareContext()`
- **No raw credential storage** — tools receive tokens at execution time only
- Brand prompt is auto-generated during onboarding but can be manually edited
- Custom workflows must pass the 12-layer AJV validation before registration
